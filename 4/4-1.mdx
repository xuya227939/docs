# docs-front 工程师开发指南

这份指南旨在帮助后端工程师和开发者快速了解如何修改和维护 `docs-front` 前端项目。

## 1. 项目概览

- **技术栈**: Next.js 15 (App Router), Bun, Tailwind CSS, TypeScript.
- **核心特性**:
    - **服务端渲染 (SSR)**: 提供更好的 SEO 支持。
    - **国际化 (i18n)**: 支持多语言 (en, zh, ja, ko, hi)。
    - **深色/浅色模式**: 支持主题切换。
    - **MDX 博客系统**: 基于文件的博客内容管理。

## 2. 目录结构

- `src/app`: 包含 App Router 的页面和布局。
    - `[lang]`: 用于本地化页面的动态路由段。
- `src/components`: 可复用的 UI 组件。
- `src/lib`: 工具函数，包括 i18n 逻辑。
- `src/config.ts`: 环境配置文件。
- `public`: 静态资源（图片、图标等）。

## 3. SEO 配置

### 全局元数据 (Global Metadata)
全局 SEO 设置（默认标题、描述、OpenGraph）定义在 `src/app/metadata.ts` 文件中。
修改 `defaultMetadata` 对象即可更新全站的默认 SEO 标签。

### 页面级元数据 (Per-Page Metadata)
每个页面（例如 `src/app/[lang]/page.tsx`）都导出一个 `generateMetadata` 函数。
要为特定页面定制 SEO：
```typescript
export async function generateMetadata({ params }) {
  // 如果需要，可以在这里获取动态数据
  return {
    title: "页面标题",
    description: "页面描述",
    // ...其他元数据
  };
}
```

## 4. 页面管理

### 路由结构
项目使用 Next.js App Router，并带有动态的 `[lang]` 路由段用于处理多语言。
- **首页**: `src/app/[lang]/page.tsx`
- **博客列表**: `src/app/[lang]/blog/page.tsx`
- **博客文章**: `src/app/[lang]/blog/[slug]/page.tsx` (隐含结构)

### 创建新页面
例如，要添加一个 `/about` 页面：
1. 创建目录: `src/app/[lang]/about`。
2. 在该目录下添加 `page.tsx` 文件。
3. 确保页面组件接收 `params: { lang: string }` 参数以处理本地化逻辑。

## 5. 头部与底部修改 (Header & Footer)

### 头部 (Header)
文件位置: `src/components/header/index.tsx`
- **Logo**: 修改 `<LogoText />` 组件或包裹它的 `Link`。
- **导航**: 在 `nav` 部分添加或删除链接。
- **逻辑**: 包含语言切换 (`useLocale`) 和主题切换的逻辑。

### 底部 (Footer)
文件位置: `src/components/footer/index.tsx`
- **内容**: 修改列布局和链接。
- **版权**: 更新年份和公司名称。
- **国际化**: 使用 `getText("footer.xxx")` 获取本地化字符串。

## 6. 国际化 (i18n)

### 配置
- **支持的语言**: 在 `src/middleware.ts` 和 `src/app/[lang]/page.tsx` 中定义 (例如 `['en', 'zh', 'ja', 'ko', 'hi']`)。
- **中间件**: `src/middleware.ts` 负责根据语言偏好进行 URL 重写和重定向。

### 翻译文件
文件位置: `src/lib/i18n/`:
- `en.json`, `zh.json` 等。
- **添加文案**: 在所有 JSON 文件中添加新的键值对。
- **使用方法**:
  ```typescript
  import { getText } from "@/lib/i18n";
  // ...
  <span>{getText("home.title")}</span>
  ```

## 7. 主题切换 (Theme Switching)

- **Provider**: 使用 `next-themes` 的 `ThemeProvider`，配置在 `src/app/layout.tsx` 中。
- **切换**: `Header` 组件中包含切换深色/浅色模式的按钮。
- **样式**: 使用 Tailwind 的 `dark:` 前缀来编写深色模式样式 (例如 `bg-white dark:bg-gray-900`)。

## 8. 博客内容管理

### 内容位置
博客文章以 Markdown/MDX 文件形式存储在：
`src/app/[lang]/blog/data/[lang]/`
(例如 `src/app/[lang]/blog/data/en/my-post.md`)

### 添加新文章
1. 在对应语言的文件夹中创建一个新的 `.md` 或 `.mdx` 文件。
2. 在文件顶部添加 **Frontmatter** (元数据):
   ```yaml
   ---
   title: "我的新文章"
   date: "2024-01-01"
   description: "文章的简短摘要。"
   tags: ["nextjs", "guide"]
   ---
   ```
3. 在 Frontmatter 下方编写正文内容。

### 修改现有内容
直接编辑对应的 `.md` 文件即可。修改后的内容会在重新构建或开发模式下（HMR 可能需要刷新）生效。
